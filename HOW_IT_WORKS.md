# Celljevity OS - How It Works

## Overview

Celljevity OS is a **Longevity Healthcare Management Platform** built with a modern serverless architecture. It allows healthcare providers to manage services, create quotes/invoices, and maintain a document vault for patients.

## Backend (Convex)

### What is Convex?

Convex is a **serverless backend platform** that provides:
- **Database** - Real-time NoSQL database with automatic indexing
- **Functions** - Serverless TypeScript functions (queries, mutations, actions)
- **File Storage** - Built-in file upload/download
- **Real-time Sync** - Automatic data synchronization to frontend
- **Hosting** - Global edge deployment

### Database Schema

#### 1. Users Table
```typescript
{
  _id: Id<"users">,           // Auto-generated by Convex
  email: string,              // Unique email address
  passwordHash: string,       // Simple hash (MVP - replace with bcrypt in production)
  name: string,               // Display name
  role: "admin" | "coordinator" | "provider",
  _creationTime: number       // Auto-generated timestamp
}
```

**Indexes:**
- `by_email` - Fast login lookups
- `by_role` - Role-based queries

#### 2. Services Table (Service Catalog)
```typescript
{
  _id: Id<"services">,
  name: string,               // "Exosome Therapy - Basic"
  description?: string,       // Service description
  price: number,              // 2500.00 (EUR)
  category: string,           // "Exosomes", "Prometheus", "NK Cells", etc.
  active: boolean,            // true/false
  _creationTime: number
}
```

**Indexes:**
- `by_category` - Filter by category
- `by_active` - Show only active services

#### 3. Quotes Table (Quotes & Invoices)
```typescript
{
  _id: Id<"quotes">,
  userId: Id<"users">,        // Creator reference
  quoteNumber: string,        // "QUO-20240316-0001"
  type: "quote" | "invoice",  // Document type
  status: "draft" | "sent" | "accepted" | "paid" | "cancelled",
  
  // Customer Info (denormalized for performance)
  customerName: string,
  customerEmail?: string,
  customerPhone?: string,
  
  // Financial Data
  subtotal: number,           // Before tax
  taxRate: number,            // 19 (default MwSt)
  taxAmount: number,          // Calculated tax
  total: number,              // Final amount
  
  notes?: string,             // Additional notes
  _creationTime: number
}
```

**Indexes:**
- `by_user` - User's quotes
- `by_type` - Filter quotes vs invoices
- `by_status` - Filter by status
- `by_quoteNumber` - Lookup by number

#### 4. QuoteItems Table (Line Items)
```typescript
{
  _id: Id<"quoteItems">,
  quoteId: Id<"quotes">,      // Parent quote
  serviceId: Id<"services">,  // Service reference
  serviceName: string,        // Snapshot (for PDF/history)
  quantity: number,
  unitPrice: number,
  total: number,
  _creationTime: number
}
```

**Indexes:**
- `by_quote` - Get all items for a quote

#### 5. Documents Table (Document Vault)
```typescript
{
  _id: Id<"documents">,
  userId: Id<"users">,        // Owner
  filename: string,           // Stored filename
  originalName: string,       // Original upload name
  mimeType: string,           // "application/pdf", etc.
  size: number,               // File size in bytes
  storageId?: Id<"_storage">, // Convex Storage reference
  category: "invoice" | "quote" | "lab-report" | "other",
  relatedQuoteId?: Id<"quotes">, // Optional link to quote
  shareToken?: string,        // Public download token
  shareExpiresAt?: number,    // Token expiry timestamp
  _creationTime: number
}
```

**Indexes:**
- `by_user` - User's documents
- `by_category` - Filter by type
- `by_quote` - Documents for a quote
- `by_shareToken` - Public access

## Backend Functions

### Authentication (`convex/auth.ts`)

#### `login`
```typescript
// Called when user submits login form
const result = await login({ 
  email: "admin@celljevity.com", 
  password: "admin123" 
});
// Returns: { userId, email, name, role } or null
```

**How it works:**
1. Query user by email index
2. Hash input password with simple hash function
3. Compare with stored passwordHash
4. Return user data (no JWT - session handled by Convex client)

#### `seedAdmin`
Creates default admin user on first run.

---

### Services (`convex/services.ts`)

#### `list`
Returns all active services for the catalog.

```typescript
const services = await list();
// Returns: [{ _id, name, description, price, category, active }]
```

#### `seedServices`
Creates 6 default services (Exosomes, Prometheus, NK Cells, Diagnostics, etc.)

---

### Quotes (`convex/quotes.ts`)

#### `list`
```typescript
const quotes = await list({ 
  userId: "...", 
  type: "quote" // optional 
});
```

#### `get`
Get single quote with all line items.

```typescript
const quote = await get({ quoteId: "..." });
// Returns quote + populated items array
```

#### `create`
Main function for creating quotes/invoices.

```typescript
const newQuote = await create({
  userId: "...",
  type: "quote",
  customerName: "Max Mustermann",
  customerEmail: "max@example.com",
  items: [
    { serviceId: "...", quantity: 2 }
  ],
  taxRate: 19
});
```

**What happens:**
1. Generate unique quote number (QUO-YYYYMMDD-XXXX)
2. Fetch service prices and calculate totals
3. Create quote record
4. Create quoteItems records
5. Return complete quote

#### `updateStatus`
Change quote status (draft → sent → accepted → paid).

---

### Documents (`convex/documents.ts`)

#### `list`
Get user's documents with optional category filter.

#### `create`
Create document record after file upload.

#### `remove`
Delete document + file from storage.

#### `getDownloadUrl` (Action)
Generate temporary download URL.

#### `validateShare`
Validate public share token for external downloads.

---

### HTTP Actions (`convex/http.ts`)

#### File Upload Endpoint
```
POST /api/uploadDocument?userId=...&category=...
Content-Type: multipart/form-data

file: <binary data>
```

**What happens:**
1. File uploaded to Convex Storage
2. Document record created with metadata
3. Share token generated (7-day expiry)
4. Returns documentId

#### File Download Endpoint
```
GET /api/downloadDocument?token=...
```

**What happens:**
1. Validate share token
2. Check expiry
3. Return file with proper headers
4. Browser triggers download

## Frontend (React)

### Architecture

**Technology Stack:**
- React 19 with TypeScript
- Vite 7 (build tool)
- Tailwind CSS (styling)
- Convex React Client (real-time data)

### Data Flow

```
┌─────────────┐     ┌──────────────────────┐     ┌──────────────┐
│  User Action │────▶│  Convex React Hook   │────▶│   Convex     │
│  (Click)     │     │  (useQuery/useMutation)│   │   Backend    │
└─────────────┘     └──────────────────────┘     └──────────────┘
                            │                             │
                            │ WebSocket                   │ Database
                            │ Update                      │ Change
                            ▼                             ▼
                     ┌──────────────┐            ┌──────────────┐
                     │  UI Updates  │◀───────────│  Real-time   │
                     │  (Reactive)  │            │  Subscription│
                     └──────────────┘            └──────────────┘
```

### Key Hooks

#### `useQuery` - Read Data
```typescript
const services = useQuery(api.services.list);
// Automatically subscribes to updates
// Re-renders when data changes
```

#### `useMutation` - Write Data
```typescript
const createQuote = useMutation(api.quotes.create);
const result = await createQuote({ ... });
// Optimistic updates + automatic cache invalidation
```

## File Storage Flow

### Upload
```
User selects file
        │
        ▼
Frontend creates FormData
        │
        ▼
POST to /api/uploadDocument
        │
        ▼
convex/http.ts → storage.store()
        │
        ▼
Create document record
        │
        ▼
Return documentId
```

### Download
```
User clicks Download
        │
        ▼
GET /api/downloadDocument?token=...
        │
        ▼
Validate token
        │
        ▼
storage.get() → blob
        │
        ▼
Return with Content-Disposition
        │
        ▼
Browser downloads file
```
